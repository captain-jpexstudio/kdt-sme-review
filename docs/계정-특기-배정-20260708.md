> 기준: 2026-07-08 데이터셋 교체(`kdefense-final-0708`) 후 solver(대상 특기) 기준 재배정 완료.
> 운영 DB 검증: 계정마다 단일 특기 정확히 300문항, 전 문항 대기(pending) 상태에서 배정.
> **계정 배부 시 위원의 특기에 맞는 계정을 전달할 것.**

## 배정 규칙 (근거)

- 데이터셋의 `solver` 컬럼(문항별 대상 특기, 7특기 × 300건)을 기준으로 1:1 배정.
- 매핑 순서 = 특기 이름 정렬순 ↔ 계정 코드 정렬순 (배정 로직 `build_assignments`와 동일 규칙).
- 향후 업로드/교체 시에도 자동 적용: `assigned_persona` 있으면 그 매핑 → 없으면 `solver` 균등 시 특기 배정 → 둘 다 아니면 랜덤.

## 예비(Reserved) 풀 — 특기별 잔여 (2026-07-08 기준)

폐기 시 **같은 특기의 예비만** 자동 대체 배정된다. 같은 특기 예비가 없으면 대체 없이 진행(작업량 감소).

## 주의

- 계정을 다른 특기 위원에게 주면 300문항 전체가 전공 불일치가 되므로, 배부 전 이 표를 반드시 확인.
- 서약(동의·서명)은 계정 단위로 기록되므로, 한 번 서약한 계정을 다른 사람에게 재배부하지 말 것.
